## Ключевые принципы системы

### 1. TDD подход (Test-Driven Development)
- **Тесты пишутся ПЕРВЫМИ**
- Затем пишется код, который проходит тесты
- Если тесты не проходят - исправление (макс 3 попытки)
- После 3 попыток - смена подхода

### 2. Самокоррекция
- Агенты сами проверяют свои результаты
- Автоматическое исправление ошибок
- Максимум 3 попытки с одним подходом
- После 3 попыток - смена подхода (другой алгоритм, архитектура)

### 3. Непрерывность
- Система работает 24/7 без остановки
- CEO генерирует задачи когда прошлые выполнены и нечем заняться
- Главный цикл никогда не останавливается

### 4. Динамические критерии
- Агенты сами определяют критерии одобрения
- Критерии зависят от задачи, проекта, роли
- Как человек: каждый раз оценивает ситуацию

### 5. Автономность
- Все решения принимаются без участия человека
- Агенты работают полностью самостоятельно

## Порядок реализации

### Этап 1: Базовая инфраструктура
1. LLM Client (интеграция с OpenAI/Anthropic)
2. Memory System (SQLite база данных)
3. Context Manager (управление контекстом)

### Этап 2: Базовые компоненты
4. Base Agent (базовый класс агента)
5. Orchestrator (оркестратор задач)
6. File System Manager (работа с файлами)

### Этап 3: Интеграции
7. Git Manager (Git операции)
8. Test Runner (запуск тестов)
9. Deployment Manager (деплой)

### Этап 4: Агенты
10. CEO Agent
11. CFO Agent
12. Developer Agent
13. QA Agent
14. Остальные агенты (по приоритетам)

## Технические требования

### Минимальные требования
- Python 3.11+
- Git
- Доступ к OpenAI API или Anthropic API

### Рекомендуемые инструменты
- Docker (для деплоя)
- CI/CD (GitHub Actions / GitLab CI)
- Мониторинг (опционально)

## Настройка окружения

1. Скопируйте `config/config.example.yaml` в `config/config.yaml`
2. Создайте `.env` файл:
   ```
   OPENAI_API_KEY=your_key_here
   # или
   ANTHROPIC_API_KEY=your_key_here
   ```
3. Установите зависимости:
   ```bash
   pip install -r requirements.txt
   ```

## Тестирование

- Начните с unit тестов для каждого компонента
- Используйте моки для LLM API
- Тестируйте интеграции отдельно
- E2E тесты для полных потоков

## Критические моменты

1. **Непрерывность**: Система должна работать всегда, даже при ошибках
2. **Обработка ошибок**: Все ошибки логируются, система не падает
3. **Таймауты**: Все операции имеют таймауты
4. **Безопасность**: API ключи в переменных окружения, не в коде
