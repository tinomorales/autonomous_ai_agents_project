# Промпт для изучения проекта

Скопируй этот промпт и используй его с ChatGPT, Claude или другим LLM для быстрого изучения проекта.

---

## Промпт

```
Я получил проект для реализации - систему полностью автономных ИИ-агентов, работающих 24/7 без участия человека.

Структура проекта:
- docs/ТЕХНИЧЕСКАЯ_АРХИТЕКТУРА.md - главный документ с детальной архитектурой
- docs/Архитектура_автономных_ИИ_агентов.md - общая архитектура
- ИНСТРУКЦИЯ_ДЛЯ_РАЗРАБОТЧИКА.md - инструкция для разработчика
- README.md - описание проекта
- requirements.txt - зависимости

Ключевые принципы системы:
1. TDD подход - тесты пишутся ПЕРВЫМИ, затем код
2. Самокоррекция - агенты проверяют и исправляют ошибки (3 попытки, потом смена подхода)
3. Непрерывность - CEO генерирует задачи когда нечем заняться
4. Динамические критерии - агенты сами определяют критерии одобрения
5. Автономность - все решения без участия человека

Основные компоненты:
- Orchestrator - координирует работу агентов
- Agent System - базовый класс и специализированные агенты (CEO, CFO, Developer, QA)
- Memory System - SQLite для хранения памяти
- File System Manager - редактирование файлов кода
- Git Manager - автоматические Git операции
- LLM Client - интеграция с OpenAI/Anthropic
- Test Runner - запуск тестов
- Deployment Manager - автоматический деплой

Помоги мне:
1. Понять архитектуру системы - объясни основные компоненты и их взаимодействие
2. Определить порядок реализации - с чего начать, какие компоненты реализовывать первыми
3. Разобраться в ключевых алгоритмах - непрерывный цикл CEO, самокоррекция агентов, TDD подход
4. Понять технические детали - структуры данных, потоки данных, конфигурация
5. Составить план реализации - разбить на этапы с конкретными задачами

Начни с объяснения общей архитектуры и основных потоков данных, затем помоги составить план реализации.
```

## Альтернативный промпт (более детальный)

```
Я разработчик, получил проект системы автономных ИИ-агентов. Мне нужно быстро разобраться и начать реализацию.

Контекст проекта:
- Система ИИ-агентов работающих 24/7 без человека
- Агенты автоматически генерируют задачи, пишут код, проверяют и исправляют ошибки
- Иерархическая структура: CEO → CFO/CTO → Операционные агенты
- TDD подход: тесты пишутся первыми
- Самокоррекция: 3 попытки исправления, потом смена подхода

У меня есть документация:
- ТЕХНИЧЕСКАЯ_АРХИТЕКТУРА.md - детальное описание всех компонентов
- Архитектура_автономных_ИИ_агентов.md - общая архитектура
- ИНСТРУКЦИЯ_ДЛЯ_РАЗРАБОТЧИКА.md - инструкция

Задачи:
1. Объясни архитектуру системы простым языком
   - Как работают агенты?
   - Как они взаимодействуют?
   - Как происходит генерация и выполнение задач?
   - Как работает самокоррекция?

2. Объясни ключевые алгоритмы
   - Непрерывный цикл CEO (как генерируются задачи)
   - Самокоррекция агента (3 попытки + смена подхода)
   - TDD подход для кода (тесты первыми)
   - Динамические критерии одобрения

3. Помоги составить план реализации
   - С чего начать?
   - Какие компоненты реализовывать первыми?
   - Какие зависимости между компонентами?
   - Как тестировать по ходу разработки?

4. Объясни технические детали
   - Структуры данных (TaskOrchestrator, BaseAgent, etc.)
   - Потоки данных между компонентами
   - Конфигурация системы
   - Интеграция с LLM API

5. Помоги с первыми шагами
   - Какую структуру папок создать?
   - Какие файлы создать первыми?
   - Как настроить окружение?
   - С чего начать кодирование?

Будь конкретным, давай примеры кода где возможно, задавай уточняющие вопросы если что-то непонятно.
```

## Как использовать

1. **Для первого знакомства**: Используй первый или второй промпт
2. **Для конкретных вопросов**: Используй третий промпт
3. **Для проверки**: Используй четвертый промпт

## Советы

- Читай документацию параллельно с использованием промпта
- Задавай уточняющие вопросы
- Проси примеры кода
- Проси объяснить сложные части простым языком
- Проверяй понимание через объяснение своими словами
